Background
INTNET was originally created by my TL in response to the Commercial Property Norms project — no existing network datasets were available in the public service at the time. Phase 1 established the EXPNET walking foundation: a 16-arc-per-node expanded grid that models HDB estate porosity (void decks, carparks, jaywalking) using the URA Masterplan 2019 land use layer. I took over in Phase 2 as the project lead.
My Contribution — Phase 2
Drove the integration of LTA Bus Routes and MRT/LRT train tracks into the walking network, unlocking true multimodal simulation. This involved snapping bus stops and cracking MRT station polygons at EXPNET intersection points, setting entry/exit transfer times (4 min in, 2 min out), and resolving topology issues across layers. Also built and delivered the full automation pipeline — Python/ArcPy scripts for data cleaning, spatial wrangling, INTNET generation.

HDB · Community Relations Group · May 2022 – Feb 2024
Project Tranquil — Community Noise Dashboard
A stakeholder-facing Python Dash dashboard enabling HDB's Community Relations Group to monitor, analyse, and act on community noise conflicts across Singapore — built on a novel Community Noise Index (CNI) derived from government-provided noise incident data.
Python Dash
4

Background
Post-Covid noise complaints spiked above CRG's amber threshold of 2.5 reports per dwelling unit. CRG needed a data-driven tool to understand noise patterns at national, town, and block levels — and to identify actionable campaign targets. Stanley designed the underlying CNI statistical methodology; I built the full interactive dashboard on top of it.
My Contribution
Built the complete Python Dash application from scratch — all four page layouts, interactive callbacks, and the data engineering pipeline that feeds them. This included a SQLite-backed data store, multi-page routing with a sidebar nav, precinct-level data fusion (joining the government-provided noise incident records to HDB building and precinct datasets), and seasonal decomposition visualisations to surface noise trends over time.
Dashboard views
[ CNI Time Series ] → Seasonally-adjusted noise index trends at national & town level
[ Top Towns ] → Year-on-year town comparison to identify highest-risk HDB towns
[ Campaign Planning · Building ] → Block-level CNI to prioritise outreach targets
[ Campaign Planning · Precinct ] → Precinct-level aggregation for area-based campaign strategy
Data pipeline
Raw noise incident records → data cleaning → CNI calculation (building / town / national) → precinct fusion → SQLite production DB → Dash callbacks → stakeholder views. Delivered with full documentation and handover to CRG for self-service reporting.
